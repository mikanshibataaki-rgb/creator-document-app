# Creator Agreement Prototype

クリエイター向け電子契約・証拠保全アプリの第1段階となる画面試作です。

## 起動

Finderから起動する場合は、次のファイルをダブルクリックします。

```text
start.command
```

自動的にGoogle Chromeで開きます。黒いターミナル画面を閉じると、アプリも停止します。

初回準備が必要な場合は、先に次のファイルをダブルクリックします。

```text
install.command
```

ターミナルから起動する場合:

```bash
npm install
npm run dev -- --hostname 127.0.0.1 --port 3100
```

ブラウザで `http://127.0.0.1:3100` を開きます。

## 現在できること

- 案件一覧の検索・状態・日付絞り込み
- 案件の新規作成と複製
- 映像制作案件の確認事項を5段階で入力
- 案件マネージャーのモックデータ取り込み
- 入力内容をブラウザ内のDB型データへ自動保存
- フリーランス法の明示候補を含む抜け漏れ確認
- 撮影順延、撮影許可、出演者・素材の権利確認
- 相手側画面、用語ヘルプ、メール確認状態の体験
- 送信者の確定、相手の同意、バージョン固定
- リマインド予定と追記専用タイムライン
- 双方共通のA4 PDFダウンロード
- PC、iPad、スマートフォン向けの表示切り替え

## 現在できないこと

- 本番用URLの発行と外部送信
- 実メール・SMS認証
- 公的な本人確認
- 認定タイムスタンプ
- サーバー・データベースへの証拠保存
- ログイン、決済、利用権管理
- 法的効力の保証

外部API、APIキー、課金サービスは使用していません。

## PDF確認

確認用PDFは次のコマンドでも生成できます。

```bash
npm run pdf:sample
```

出力先:

```text
output/pdf/creator-agreement-sample.pdf
```
